# NREL NSRDB Raw Data

This directory is intentionally empty in git. The `.gitignore` at the repo
root excludes everything under `data/raw/` because the NSRDB CSVs are
hundreds of MB per year and we do not want them in version control.

## What Goes Here

Hourly solar irradiance and meteorological CSVs from the NREL National
Solar Radiation Database (NSRDB). One CSV per station-year.

Recommended naming convention:

```
data/raw/nrel_solar/<station_id>_<year>.csv
```

For example: `data/raw/nrel_solar/722874_2022.csv`.

## Expected Columns

The download script (`scripts/download_nrel_data.py`, not yet implemented
in this branch) normalizes NSRDB physical files into the following columns
that `services/ml/data_prep.py::load_and_prepare` expects:

| column              | type    | units  | description                       |
|---------------------|---------|--------|-----------------------------------|
| `timestamp`         | str     | ISO    | `"YYYY-MM-DD HH:MM:SS"`           |
| `temperature_c`     | float   | °C     | air temperature                   |
| `cloud_cover_pct`   | float   | %      | 0-100                             |
| `solar_irradiance`  | float   | W/m²   | GHI (Global Horizontal Irradiance)|

`solar_generation_kw` (the target column) is **not** in the raw file — it is
derived from `solar_irradiance` and a system capacity constant at load time,
or it is replaced with measured generation if a co-located meter log exists.

## How to Get the Data

1. Create a free NREL API key: https://developer.nrel.gov/signup/
2. From the repo root, run:
   ```bash
   poetry run python scripts/download_nrel_data.py \
       --station 722874 \
       --year 2022 \
       --email your@email.com \
       --api-key $NREL_API_KEY \
       --out data/raw/nrel_solar/
   ```
3. Verify the file landed and has the expected columns:
   ```bash
   poetry run python -c "import pandas as pd; print(pd.read_csv('data/raw/nrel_solar/722874_2022.csv').head())"
   ```
4. Retrain the model against the real data:
   ```bash
   poetry run python -m services.ml.train --data-path data/raw/nrel_solar/722874_2022.csv
   ```

## For Development Without Real Data

If you do not have an NREL API key (or do not want to download several
hundred MB of CSVs), the training pipeline falls back to a synthetic
dataset generated by `services.ml.data_prep.generate_synthetic_data`.
The synthetic data follows the same column schema. Models trained on
synthetic data are useful for plumbing tests but should not be used for
real grid optimization — the numbers are plausible, not real.
